# Reproducing the Paper Artifacts

This document records the commands used to regenerate the reproducibility
artifacts referenced by `paper.tex`.  Run commands from the repository root.

## Environment

Required tools:

- Python 3.10+ for scripts under `notes/scripts/`
- A LaTeX distribution with `latexmk` or `xelatex`
- Lean 4 / Lake for the formalization under `lean/`

No Python package is required for the deployment-parameter table added in this
branch; it uses only the Python standard library.

## Deployment Parameter Tables

Issue #356 asks for final non-interactive deployment parameters.  Regenerate the
table with:

```bash
python3 notes/scripts/deployment_params.py
python3 notes/scripts/deployment_params.py --latex
python3 notes/scripts/deployment_params.py > notes/scripts/deployment_params.output.txt
```

The script implements the Appendix D bounds:

```text
interactive:     eps <= nR/|F| + (1 - delta/2)^q
non-interactive: eps <= Q * eps_interactive + 3(Q^2 + 1)/2^kappa
```

The default configuration uses `n=2^20`, `R=20`, `delta=0.4`,
`q=128`, `Q=2^64`, and SHA-style `kappa=256` except for the
`BabyBear7` row, where `kappa=384` is used so the hash-binding term is
not the bottleneck for the 128-bit example.

To reproduce the optional tighter Fiat-Shamir accounting mentioned in
Remark D.2, run:

```bash
python3 notes/scripts/deployment_params.py --per-round-max
python3 notes/scripts/deployment_params.py --per-round-max --latex
```

## Lean Formalization

The in-repository Lean files live under `lean/`.  Build them with:

```bash
cd lean
lake build
```

Current paper wording distinguishes the proved Lean core from remaining work:
the half-threshold CA bound, the coupling chain, and batch CA are formalized;
the RS isomorphism axioms and probability-level soundness theorem remain open
work tracked separately in issue #350.

## Existing Computational Outputs

Several scripts in `notes/scripts/` have checked-in `.output.txt` files next to
the corresponding script.  These files are intended as saved audit outputs.  To
refresh one of them, run the script and redirect stdout to the matching output
file, for example:

```bash
python3 notes/scripts/op1_scaling.py > notes/scripts/op1_scaling.output.txt
python3 notes/scripts/op1_scaling_n8.py > notes/scripts/op1_scaling_n8.output.txt
python3 notes/scripts/audit_fri_coupling.py > notes/scripts/audit_fri_coupling.output.txt
```

Some exploratory scripts are expensive or require GPU-oriented environments.
Do not treat every script in `notes/scripts/` as part of the lightweight
reproduction path.  The paper should cite a script/output pair only when the
result is either reproduced by the commands above or explicitly marked as
computational evidence.

## Building the Paper

Build the PDF with:

```bash
latexmk -pdf paper.tex
```

If `latexmk` is unavailable, use:

```bash
xelatex paper.tex
xelatex paper.tex
```

The tracked `paper.pdf` should be regenerated only when intentionally updating
the rendered paper artifact.
